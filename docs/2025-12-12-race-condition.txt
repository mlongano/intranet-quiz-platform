  The Problem: Read-Modify-Write Race

  While save_scores() uses file locking (utils.py:172-207), the load happens
   OUTSIDE the lock, creating a classic read-modify-write race condition:

  Timeline with 2 concurrent submissions:

  Time    Student A                           Student B
  ─────────────────────────────────────────────────────────────────
  T0      load_scores() → [previous_scores]
  T1                                          load_scores() → 
  [previous_scores]
  T2      grading... (takes time)
  T3                                          grading... (takes time)
  T4      append → [previous_scores, A]
  T5      save_scores([previous_scores, A])   ← Saves successfully
  T6                                          append → [previous_scores, B]
  T7                                          save_scores([previous_scores,
  B]) ← OVERWRITES!

  Result: Student A's score is LOST because:
  1. Student B loaded the scores BEFORE Student A saved
  2. Student B's list doesn't include Student A
  3. Student B overwrites the entire file

  Why the Lock Doesn't Help

  The lock in save_scores() only prevents simultaneous writes, but the
  damage is already done because:
  - Both students read the OLD scores before either saved
  - Each builds their own complete list
  - Whoever saves last wins, losing the other's data

  The Fix

  The load-append-save sequence must happen atomically within the same lock:

  # BAD (current code):
  scores = load_scores()              # Outside lock
  scores.append(new_score)
  save_scores(scores)                 # Inside lock

  # GOOD (proposed fix):
  # Move the entire read-modify-write inside a single locked transaction

  Would you like me to implement a fix for this race condition? The solution
   would involve:

  1. Moving the lock higher - Acquire the lock BEFORE loading scores
  2. Atomic transaction - Keep the lock for the entire read-modify-write
  sequence
  3. Retry logic - Handle lock timeout gracefully

  This explains why your user saw success but their score wasn't saved -
